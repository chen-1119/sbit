# 小程序搜索关键词与内容规划

## 1. 关键词策略

微信小程序搜索不是传统网页 SEO，代码里没有可配置的 meta keywords。当前项目能做的优化是：

- 在首页真实展示核心关键词，避免隐藏词堆砌。
- 使用清晰页面标题、分享标题和模块标题。
- 通过 `sitemap.json` 允许首页、主测和模块入口被索引，屏蔽个人结果页。
- 模块入口尽量使用可直达的 `navigator` 路径，减少纯脚本跳转对爬虫的依赖。

建议在公众平台后台同步配置：

- 小程序名称：`sbti人格趣味调侃`
- 简介建议：`MBTI、DISC、九型人格、霍兰德、爱之语言、EQ等趣味人格测试合集`
- 核心词：`人格测试`、`MBTI测试`、`DISC测试`、`九型人格`、`霍兰德职业兴趣`
- 长尾词：`爱之语言测试`、`EQ情绪能力测试`、`职业性格测试`、`恋爱关系测试`、`麋鹿测试`

## 3. 建议新增测试

优先级从产品价值和搜索需求综合考虑：

1. 依恋风格测试
   关系安全感、焦虑型、回避型、安全型，适合承接关系类搜索。
2. 职业价值观测试
   收入、自由、稳定、成长、影响力、生活平衡，适合承接职业规划搜索。
3. 学习行动力测试
   学习风格、拖延触发点、反馈偏好，适合承接学生和成长类用户。
4. 社交边界感测试
   边界、拒绝能力、社交能量、关系负荷，适合承接沟通和关系类用户。

## 4. 结果展示优化方向

结果页应避免只给一个标签，推荐固定展示：

- 主导维度：告诉用户最明显的倾向。
- 次高维度：解释为什么同一类型的人表现会不同。
- 清晰度：说明结果是集中型还是混合型。
- 行动建议：给 1-3 条可执行建议。
- 下一步测试：引导用户继续完成相关模块，提升留存和分享。

## 5. 注意事项

- 不复制商业量表原题，只参考公开维度结构。
- 不承诺医学、临床、招聘或重大职业决策准确性。
- 不在页面堆不可见关键词，避免爬虫看到的内容和真实用户看到的内容不一致。
