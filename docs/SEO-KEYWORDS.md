# 小程序搜索关键词与内容规划

## 1. 关键词策略

微信小程序搜索不是传统网页 SEO，代码里没有可配置的 meta keywords。当前项目能做的优化是：

- 在首页真实展示核心关键词，避免隐藏词堆砌。
- 使用清晰页面标题、分享标题和模块标题。
- 通过 `sitemap.json` 允许首页、主测和模块入口被索引，屏蔽个人结果页。
- 模块入口尽量使用可直达的 `navigator` 路径，减少纯脚本跳转对爬虫的依赖。

建议在公众平台后台同步配置：

- 小程序名称：`sbti人格趣味调侃`
- 简介建议：`MBTI、DISC、九型人格、霍兰德、爱之语言、EQ等趣味人格测试合集`
- 核心词：`人格测试`、`MBTI测试`、`DISC测试`、`九型人格`、`霍兰德职业兴趣`
- 长尾词：`爱之语言测试`、`EQ情绪能力测试`、`职业性格测试`、`恋爱关系测试`、`依恋风格测试`、`社交边界感测试`、`职业价值观测试`、`学习行动力测试`、`麋鹿测试`

## 3. 已落地的新增测试

本轮已按产品价值和搜索需求落地以下模块：

1. 依恋风格测试
   代码模块：`attachment_security`。覆盖关系安全感、焦虑追问、距离防御、矛盾拉扯。
2. 职业价值观测试
   代码模块：`career_values`。覆盖自由、成长、稳定、收益、影响力、生活平衡。
3. 学习行动力测试
   代码模块：`learning_drive`。覆盖计划推进、探索输入、互动反馈、冲刺激活、复盘沉淀。
4. 社交边界感测试
   代码模块：`social_boundary`。覆盖清晰边界、弹性协商、过度承担、高墙防御。

## 4. 后续可扩展测试

- 友情人格测试：朋友关系、回应期待、玩笑边界和社交默契。
- 消费人格测试：情绪消费、理性预算、体验优先和囤货倾向。
- 城市生活方式测试：居住节奏、通勤忍耐、夜生活、自然需求和机会密度。
- 恋爱沟通雷区测试：冷处理、追问、讲道理、试探和修复方式。

## 5. 结果展示优化方向

结果页应避免只给一个标签，推荐固定展示：

- 主导维度：告诉用户最明显的倾向。
- 次高维度：解释为什么同一类型的人表现会不同。
- 清晰度：说明结果是集中型还是混合型。
- 朋友视角：方便截图和转发时让朋友参与验证。
- 适用场景：告诉用户这个结果在哪些环境更容易发挥。
- 避坑提醒：给出一个最低成本的行为修正点。
- 城市匹配：把抽象气质转成更容易传播的城市建议。
- 行动建议：给 1-3 条可执行建议。
- 下一步测试：引导用户继续完成相关模块，提升留存和分享。

## 6. 注意事项

- 不复制商业量表原题，只参考公开维度结构。
- 不承诺医学、临床、招聘或重大职业决策准确性。
- 不在页面堆不可见关键词，避免爬虫看到的内容和真实用户看到的内容不一致。
